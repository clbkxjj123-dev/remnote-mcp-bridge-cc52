# RemNote Notes

将学习材料转换为来源忠实、检索目标明确、可直接粘贴进 RemNote 的笔记草稿。把生成结果视为待预览的学习设计，不把 AI 生成卡片视为无需复核的成品。

## 必读资源

- 每次生成或修复可粘贴文本前，完整读取 [references/syntax-full.md](references/syntax-full.md)。它区分文本导入语法与编辑器命令。
- 每次设计或审查闪卡前，完整读取 [references/card-quality.md](references/card-quality.md)。它包含来源、原子性、方向、去重和质量门槛。
- 会话中存在 `remnote_*` MCP 工具（RemNote MCP Bridge）时，完整读取 [references/mcp-bridge.md](references/mcp-bridge.md)，改用 MCP 直写交付；无桥接时跳过该文件。
- 仅在用户明确要求标签、Reference、Portal、Universal Descriptor、代码块、公式或图片时，读取语法参考中的对应小节；不要凭其他笔记软件的语法推断 RemNote 行为。

## 执行流程

### 1. 确定学习任务与交付模式

识别材料范围、学习目标、考试或应用情境、希望保留的细节和卡片预算。材料已足够时直接执行；只有缺少材料、目标相互冲突或关键歧义会显著改变结果时才询问。

同时判定交付模式：会话有 `remnote_*` MCP 工具且 `remnote_status` 确认已连接时，默认 MCP 直写（工具细节见 mcp-bridge.md）；否则输出可粘贴正文。用户明确指定模式时以用户为准。

用户未给预算时采用保守策略：先覆盖核心概念与先修关系；每个核心概念通常保留 1 个定义或识别目标，再按价值增加 0–2 个机制、边界、对比或应用目标。不要为了减少卡数而合并独立检索目标。

### 2. 建立证据账本

通读材料并把候选知识分成三层：

1. **原文明示**：可直接定位到材料中的事实、关系或限定条件。
2. **严格推导**：不引入新事实即可从材料推出的结论。
3. **外部补充**：依赖材料之外的知识。

默认只为前两层制卡。不得强化或删除材料中的数值、范围、否定、概率、比较、因果和不确定性限定词。信息含糊、互相冲突或缺少前提时，降为普通参考笔记或列入待核实项，不猜测答案。MCP 直写模式下，待核实项可落为 RemNote 原生 TODO（`set_rem_features`，见 mcp-bridge.md 第 9 节），让用户复习时可直接处理。

仅在用户允许外部补充时加入第三层；逐条标记“外部补充”并给出可核验来源。不要把外部补充伪装成材料原意。

### 3. 筛选记忆价值

按以下顺序取舍：

- **必记**：学习目标直接要求、后续推理依赖、错误代价高或高频应用的内容。
- **选记**：典型对比、关键例子、常见边界或有区分力的细节。
- **参考**：有检索价值但不值得定期回忆的背景，保留为无卡笔记或禁用 Descriptor。
- **跳过**：重复、过渡、闲聊、装饰性细节和无法可靠判定的内容。

忠实而原子化的改写本身就是合格卡片；不要强制每张卡都“超出原文”，也不要为制造理解增量而编造事实。

### 4. 定义检索目标

先为每张候选卡写内部检索目标：`对象 | 关系 | 条件 | 方向`。一个实际复习方向只测试一个能独立评分的目标。

出现多个可分别评分的事实、多个独立从句，或要求“列出并解释”时拆卡。答案保留客观判分所需的最小充分信息；短不是目的，可稳定评分才是目的。

### 5. 选择卡型与方向

- 用 `:>` 表示只需“名称 → 定义”的 Concept；只有定义能唯一识别名称且反向检索有现实价值时才用 `::`。
- 用缩进的 `;;` 表示 Concept 的单一属性或问题。使用普通、稳定的 Descriptor 名称，如“适用条件”“作用”“主要区别”。
- 用 `>>` 表示因果、比较、关系、判断理由和情境应用。
- 少量使用 `{{答案}}` 测试适合精确补全的词、值或固定文本；上下文不得靠措辞或语法泄露答案。
- 只有必须完整回忆一个无序集合时使用 Set `>>>`；只有顺序本身必须记忆时使用 List `>>1.`。
- 只有用户准备选择题考试或提供题库时才使用 `>>A)`；一般学习优先开放回忆。
- 用 `;-` 保留 Concept 下有检索价值但不需要复习的属性。
- 材料含系统性多维对比（≥3 个同类概念 × ≥2 个稳定维度）且处于 MCP 直写模式时，建 Advanced Table 作概览层（mcp-bridge.md 第 8 节）；卡片只测有区分力或易混的单元格，不做"每格一卡"。粘贴模式没有表格能力，改用普通 Descriptor 分列陈述。

不要把 `~作用 ;;` 等文本称为真正的 Universal Descriptor。除非用户确认知识库已创建对应 Universal Descriptor Rem 或 Property，否则默认输出普通 Descriptor 名称。

Concept 命名与元数据遵守 card-quality.md 的"命名与元数据规范"：主名称用稳定专业术语、不混入数学符号或英文名；英文术语放 alias（仅 MCP 模式可直接写入）；数学记号放 `~常用记号 ;-`、定义或公式描述符。

### 6. 去重与增强理解

比较本批次所有规范化检索目标。相同对象、关系、条件和方向只保留最清晰的一张；Concept、Basic 和 Cloze 测同一事实时同样去重。不要把同一命题拆成“对哪类有效？”和“对哪类无效？”等互补问法，除非两个边界都必须独立检索且单问无法覆盖。

仅当检索目标确实不同，才保留同一事实的第二张卡，例如定义回忆与情境应用。对每个核心机制最多补充少量高价值的 why/how、边界、对比或应用卡；答案必须由材料明示或严格蕴含。

不要默认输出 `[[名称]]`、标签或 Portal 语法来“增强连接”。纯文本粘贴不保证把它们解析为既有 Rem；改在输出后的人工操作清单中列出建议连接。MCP 直写模式例外：可先用 `remnote_search` 查重并取得 remId，再以 `[[id:<remId>]]` 建立精确引用（规则见 mcp-bridge.md 第 5 节）。

### 7. 生成正文

两种模式共用同一套官方文本导入分隔符（粘贴正文与 MCP `content` 的语法差异见 mcp-bridge.md 第 3 节）。采用 4 个空格表示每层缩进，并保持全篇一致。默认不加 Markdown 行首 `-`，避免列表符号与卡片内容混淆。

不要在可粘贴正文中使用 `/hint`、`/extra`、`/ecd`、`((Portal))`、`{{[[Portal]]}}` 或假定能自动解析的 Wiki-link。Cloze 内联提示和 Extra Card Detail 的文本导入例外见语法参考。MCP 模式下 `/extra` 子行是管道原生支持的例外；`/hint` 在两种模式都不写入正文（MCP 用 `frontCardHint`/`backCardHint` 参数，见 mcp-bridge.md 第 6 节）。

复杂 Concept 的多行答案不要使用官方文档中存在歧义的高级 Concept/Descriptor 分隔符；改为普通 Concept，再在其子级创建可靠的 Basic Set 或 List 卡。

### 8. 运行质量门槛

逐卡执行 [references/card-quality.md](references/card-quality.md) 的硬门槛。来源忠实、原子性、题干特异性、答案可判分或非重复性任一失败，就重写、降为参考笔记或删除。

正文已写入文件或可通过标准输入传递时，运行：

```powershell
python scripts/check_remnote_text.py <output.txt>
# 或从标准输入读取
Get-Content <output.txt> -Raw | python scripts/check_remnote_text.py -
```

修复所有 `ERROR`。审视每个 `WARNING`；只有确认属于有意用法时才保留。若无法运行脚本，按语法参考末尾的人工检查表逐项检查。MCP 直写的 `content` 用 `--mcp` 标志检查（放行 `[[id:<remId>]]` 与 `/extra` 子行，仍拦截 `/hint`）。

### 9. 输出与交付

**MCP 直写模式**：按 mcp-bridge.md 第 4 节执行——查重 → 建树（`asDocument`/`tagRemIds` 随建随挂）→ 增量插入 → 增强（hint/alias/标签/对比表格/代码语言/TODO/引用块/来源链接）→ 学习日志（可选，第 10 节）→ 回读验收；先试写 1 个最小样例回读确认，再写全量。交付内容改为：写入位置与 remIds、核对摘要、卡片统计、回读验收结果（区分"API 已验证"与"UI 待用户确认"）、待核实项。

已有笔记的增强不重建：普通 Rem 升级为闪卡用 `update_note.backText`（事后制卡，第 6 节）；结构调整用 `move_note`/`set_document_status` 的 dryRun-first 流程（第 9 节）。

**粘贴模式**按以下结构输出；没有对应内容时省略该节：

1. **可粘贴正文**：单个 `text` 代码块；代码块内不放解释、注释、统计或未支持语法。
2. **核对摘要**：说明覆盖范围、取舍、假设与来源层级。
3. **卡片统计**：分别报告“产卡 Rem 数”和“初始复习方向或 Cloze 遮挡数”。`::` 按 2 个方向计算，`:>` 与 `;;` 按 1 个方向计算，每个独立 Cloze 按 1 个遮挡计算。
4. **导入后操作**：列出建议创建的 Reference、Universal Descriptor、普通卡提示、Portal、图片或其他必须在编辑器中完成的操作。
5. **待核实项**：列出材料冲突、缺失前提或未获准核验的外部事实。

不要声称生成结果已经在 RemNote 中验证。提醒用户粘贴后抽样预览至少一个 Basic、Concept/Descriptor、Cloze、Set/List 和 Extra Card Detail 实例；出现解析差异时以当前 RemNote Help 与应用预览为准。

## 材料适配

- **教材、PDF、文章**：保留章节层级；把粗体或术语视为候选 Concept，而非自动制卡指令；把图表结论转换为可判分关系。
- **视频字幕**：删除口头填充与重复；保留改变含义的例子和限定条件；时间戳通常不输出。
- **截图标记**：把截图文件名保留为“需手动插图”的普通文本占位，不宣称已经导入图片。
- **数学与代码**：优先测试适用条件、变量含义、关键转折和选择理由；不要默认背诵整段证明、流程或代码。公式与代码的导入限制见语法参考。MCP 模式下代码 Rem 建后用 `set_rem_features` 设 `codeLanguage` 获得语法高亮；材料原文引用可标 `isQuote` 与改写内容区分。
- **历史与人文**：保留时间范围、主体和立场限定；区分事实、解释与论点，不把争议性解释写成确定事实。

## 最小示例

材料：“二分查找适用于有序数组。每次比较中间元素，并把搜索范围缩小一半，因此时间复杂度为 O(log n)。”

```text
二分查找 :> 在有序数组中反复比较中间元素、每次将搜索范围缩小一半的查找方法
    适用条件 ;; 数组有序
    时间复杂度 ;; O(log n)
为什么二分查找的时间复杂度是 O(log n)？ >> 每次比较都将剩余搜索范围缩小一半
```

该示例包含 4 个产卡 Rem、4 个初始复习方向；不创建未经证实的 Reference，也不增加材料之外的事实。
